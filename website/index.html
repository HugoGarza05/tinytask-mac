What It Does

A native macOS workflow for loops, relogs, and focused playback.

Built for long sessions where a single loop is not enough. TinyTaskMac handles the repeatable path, the timed interruption, and the return to the main routine.

TL

Target Lock

Keep macro playback attached to the intended app so the desktop does not receive stray input.

FP

Focus Protection

Optionally pull the target app back to the front if another window covers it during playback.

PA

Preferred App Override

Save a preferred playback app globally and let it override each macro’s recorded target when needed.

RT

Routine Scheduler

Run one main loop forever and insert periodic interrupt macros by timer and priority.

Timed Interrupt Routines

Pause the grind, run the recovery flow, then start the main macro again.

Use one macro as the main loop and schedule supporting macros for relogging, memory-reset patterns, or other periodic maintenance. Higher-priority interrupts run first when multiple timers line up.

Permission Setup

TinyTaskMac now has a dedicated setup flow for Accessibility and Input Monitoring, with direct buttons to the right macOS privacy panes and live re-checks before recording or playback begins.

FAQ

What launch users need to know.

Is macOS supported right now?

Yes. TinyTaskMac is built for macOS first, with website downloads and packaged releases centered on Mac.

Can it handle long grinding sessions?

Yes. The current feature set is designed around reliable loop playback, target focus control, and timed interrupt routines.

Can a relog macro interrupt the main loop?

Yes. Routines can run one main macro continuously and insert higher-priority timed interrupts that restart the main loop afterward.

Why does it need permissions?

macOS requires Accessibility to send synthetic input and Input Monitoring to record your own inputs for new macros.

Are Windows and Linux available?

Not yet. The site tracks those platforms as planned so Mac users get the first stable release path.
